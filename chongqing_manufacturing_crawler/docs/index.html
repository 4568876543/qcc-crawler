📖 项目简介

企查查行业搜索爬虫是一个基于 Playwright 的智能数据采集工具，用于从企查查网站爬取制造业各细分行业在各区县的企业数量分布数据。

适用于经济分析、产业研究、市场调研等场景，帮助用户快速获取区域产业布局信息。

async def init_browser(self):
    # 尝试加载已保存的 Cookie
    if os.path.exists(COOKIE_FILE):
        with open(COOKIE_FILE, 'r') as f:
            cookies = json.load(f)
        await self.context.add_cookies(cookies)

    # 检查登录状态
    if await self.check_login():
        return True

    # 未登录，等待用户扫码
    await self.wait_for_login()

async def save_cookies(self):
    cookies = await self.context.cookies()
    with open(COOKIE_FILE, 'w') as f:
        json.dump(cookies, f)  # 保存到文件

3. 断点续爬核心 - 任务状态管理

utils/task_manager.py - 任务状态持久化
class TaskManager:
    def _load(self):
        # 从磁盘加载任务状态
        with open(self.task_file, 'r') as f:
            data = json.load(f)
        self.session = CrawlSession(**data['session'])
        self.industry_tasks = {
            code: IndustryTask(**task)
            for code, task in data['industry_tasks'].items()
        }

    def get_pending_industries(self):
        # 获取待处理行业（包括中断后未完成的）
        return [t for t in self.industry_tasks.values()
                if t.status in [PENDING, FAILED, IN_PROGRESS]]

    def complete_industry(self, code, district_count, count):
        self.industry_tasks[code].status = COMPLETED
        self.industry_tasks[code].enterprise_count = count
        self.save()  # 持久化到磁盘

4. 数据校验 - 防止行业选择未生效

crawler_changsha.py - 关键的数据合理性校验
async def crawl_single_industry(self, industry_code, industry_name):
    # 点击行业
    if await self.click_industry(industry_name):
        district_data = await self.get_district_distribution()
        industry_total = sum(district_data.values())

        # ===== 数据合理性校验 =====
        city_total = sum(self.city_total_data.values())
        ratio = industry_total / city_total

        if ratio > 0.9:  # 超过90%视为异常
            log(f"⚠️ 数据异常: {industry_name} 接近制造业合计")
            # 自动重试
            await self.deselect_industry(industry_name)
            if not await self.click_industry(industry_name):
                return False
            district_data = await self.get_district_distribution()

5. 筛选条件强制验证

验证三个关键筛选条件
async def verify_filters_are_set(self) -> bool:
    page_text = await self.page.text_content('body')

    # 检查地区
    location_ok = self.search_location in page_text

    # 检查登记状态
    status_ok = self.company_status in page_text

    # 检查制造业
    manufacturing_ok = "制造业" in page_text

    if not (location_ok and status_ok and manufacturing_ok):
        log(f"❌ 筛选条件验证失败")
        return False
    return True

📱 功能特点

1. 智能登录

支持扫码登录和 Cookie 自动持久化，首次登录后自动保存凭证，后续运行无需重复登录。

2. 断点续爬

程序采用 JSON 文件存储任务状态，包含：

  • task_state.json - 任务进度状态
  • temp_data.json - 临时数据缓存
  • qcc_cookies.json - 登录凭证

3. 数据校验机制

⚠️ 关键设计

如果某行业数据 > 制造业合计的 90%，系统判定为异常，自动重试行业选择。这是防止"行业选择未生效"导致的数据错误。

4. 筛选条件验证

在爬取前自动验证三个关键筛选条件：

  • ✅ 地区是否正确选择
  • ✅ 登记状态是否勾选
  • ✅ 制造业行业是否点击

5. 执行进度监控

Excel 文件第一个工作表实时显示：

  • 总行业数、已完成、进行中、待处理、失败数量
  • 所有行业的状态（颜色区分）
  • 待爬取行业列表（橙色标注）
  • 失败行业列表（红色标注）

⚠️ 常见问题

1. 登录失败

解决：删除 data/qcc_cookies.json，重新扫码登录

2. 行业选择未生效

现象：某行业数据接近制造业合计（异常）

解决：程序已自动重试，检查截图 WARNING_*_data_anomaly.png

3. 筛选条件验证失败

解决：检查 logs/screenshots/FILTER_VERIFICATION_FAILED.png，确保页面正常加载

4. 断点续爬失效

解决：检查 data/task_state.json 和 temp_data.json 是否存在且完整

💡 提示：定期查看 logs/changsha_crawler.log 了解程序运行状态

📊 数据质量保障

程序通过多重机制保障数据质量：

  • 行业选择验证：点击后检查页面状态，确保选择生效
  • 数据合理性校验：单项数据不超过合计的90%
  • 交叉验证：城市汇总 vs 细分行业明细对比
  • 截图留痕：每个关键步骤都有截图存档
  • 临时数据持久化：每完成一个行业立即保存
✅ 质量标准：城市汇总与细分行业明细差异 < 2% 视为合格

🔧 开发者指南

添加新的筛选条件

在 config_changsha.py 中添加配置项，然后在 setup_filters() 中实现对应的选择逻辑。

扩展数据校验规则

在 data_validator.py 中添加新的校验方法，或在 crawl_single_industry() 中添加自定义校验逻辑。

调试技巧
