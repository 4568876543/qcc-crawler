📖 项目简介

企查查行业搜索爬虫是一个基于 Playwright 的智能数据采集工具，用于从企查查网站爬取制造业各细分行业在各区县的企业数量分布数据。

适用于经济分析、产业研究、市场调研等场景，帮助用户快速获取区域产业布局信息。

核心价值

🚀 自动化采集

全自动流程，从登录到数据输出无需人工干预

💾 断点续爬

程序中断后可从上次位置继续，数据不丢失

✅ 数据校验

自动检测异常数据，确保数据准确性

📊 多维输出

城市汇总、行业明细、执行进度一目了然

🛠️ 技术栈

Python 3 Playwright pandas openpyxl asyncio JSON

关键技术点

  • Playwright 异步 API：高效的浏览器自动化控制
  • Cookie 持久化：登录状态自动保存，无需重复扫码
  • 任务状态管理：JSON 文件存储任务进度，支持精确续爬
  • 数据一致性校验：城市汇总 vs 细分行业交叉验证
  • 异常自动重试：行业选择失败自动重试机制
  • 实时进度监控：Excel 内置执行进度表

📱 功能特点

1. 智能登录

支持扫码登录和 Cookie 自动持久化，首次登录后自动保存凭证，后续运行无需重复登录。

2. 断点续爬

程序采用 JSON 文件存储任务状态，包含：

  • task_state.json - 任务进度状态
  • temp_data.json - 临时数据缓存
  • qcc_cookies.json - 登录凭证

3. 数据校验机制

⚠️ 关键设计

如果某行业数据 > 制造业合计的 90%，系统判定为异常，自动重试行业选择。这是防止"行业选择未生效"导致的数据错误。

4. 筛选条件验证

在爬取前自动验证三个关键筛选条件：

  • ✅ 地区是否正确选择
  • ✅ 登记状态是否勾选
  • ✅ 制造业行业是否点击

5. 执行进度监控

Excel 文件第一个工作表实时显示：

  • 总行业数、已完成、进行中、待处理、失败数量
  • 所有行业的状态（颜色区分）
  • 待爬取行业列表（橙色标注）
  • 失败行业列表（红色标注）

🔄 工作流程

1
初始化浏览器
加载 Cookie 或扫码登录，检查登录状态
2
设置筛选条件
地区 + 登记状态（存续）+ 制造业
3
验证筛选条件
确认地区/状态/行业都已正确设置，否则报错退出
4
爬取城市汇总
获取制造业在各区县的合计数据
5
遍历细分行业
逐个点击31个细分行业，获取各区县数据（自动校验+重试）
6
保存与验证
生成Excel、截图，数据一致性验证

⚠️ 常见问题

1. 登录失败

解决：删除 data/qcc_cookies.json，重新扫码登录

2. 行业选择未生效

现象：某行业数据接近制造业合计（异常）

解决：程序已自动重试，检查截图 WARNING_*_data_anomaly.png

3. 筛选条件验证失败

解决：检查 logs/screenshots/FILTER_VERIFICATION_FAILED.png，确保页面正常加载

4. 断点续爬失效

解决：检查 data/task_state.json 和 temp_data.json 是否存在且完整

💡 提示：定期查看 logs/changsha_crawler.log 了解程序运行状态

📊 数据质量保障

程序通过多重机制保障数据质量：

  • 行业选择验证：点击后检查页面状态，确保选择生效
  • 数据合理性校验：单项数据不超过合计的90%
  • 交叉验证：城市汇总 vs 细分行业明细对比
  • 截图留痕：每个关键步骤都有截图存档
  • 临时数据持久化：每完成一个行业立即保存
✅ 质量标准：城市汇总与细分行业明细差异 < 2% 视为合格
