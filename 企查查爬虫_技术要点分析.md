## 四、爬虫主流程 (crawler_changsha.py)

### 4.1 主要流程

```
1. init_browser()           # 启动浏览器（支持Cookie持久化）
2. navigate_to_search()     # 访问企查查首页
3. check_login()            # 检查登录状态（等待扫码登录）
4. setup_filters()          # 设置筛选条件（省份、城市、状态）
5. crawl_all_industries()   # 点击"制造业"，获取各区县汇总数据
6. crawl_industries_batch() # 遍历31个细分行业，获取各区县分布
7. save_summary_table()     # 保存汇总明细表
8. validate_data_consistency() # 数据一致性验证
```

### 4.2 setup_filters() 方法（第341-452行）- 关键问题位置

```python
async def setup_filters(self):
    """设置筛选条件"""
    # ...
    # 问题1: 第366-380行 - 选择湖南省
    self.log("  点击湖南省...")
    try:
        hunan_btn = self.page.locator('text=湖南省').first

    except Exception as e:
        self.log(f"  点击湖南省失败: {e}")  # 异常被捕获，继续执行

    # 问题2: 第382-392行 - 选择长沙市
    self.log(f"  点击{CITY}...")  # CITY = "湖南省" 但这里用的是"长沙市"
    try:
        city_btn = self.page.get_by_text(CITY).first  # ⚠️ 可能找不到"湖南省"
        await city_btn.click()
    except Exception as e:
        self.log(f"  点击{CITY}失败: {e}")
```

### 4.3 省份选择问题分析

**问题现象**: 查找省一级的时候，没有选择湖南省

**问题原因**:
1. `CITY = "湖南省"` 与实际想要选的"长沙市"不符
2. 选择省份使用 `self.page.locator('text=湖南省').first` 可能选择器不正确
3. 点击失败后异常被捕获但程序继续执行
4. 没有对省份选择结果进行验证

**页面交互流程**:
```
1. 输入"湖南省"搜索 → 进入湖南省企业列表页
2. 点击"更多"展开省份地区筛选
3. 在省份列表中点击"湖南省" → 筛选湖南省
4. 在城市列表中点击"长沙市" → 筛选长沙市
5. 设置登记状态为"存续/在业"
```

## 九、断点续爬机制

- 临时数据保存到 `temp_data.json`
- 任务状态保存到 `task_state.json`
- 每次完成一个行业自动保存
- 程序意外退出后重启会自动检测并继续

## 十、技术债务/问题

1. **配置硬编码**: 城市、省份、关键字都是硬编码
2. **异常处理过于宽松**: 点击失败后继续执行可能导致后续问题
3. **选择器脆弱**: 使用 `text=湖南省` 这种文本匹配不够稳定
4. **无UI自动化测试**: 没有对页面交互进行自动化测试
5. **浏览器上下文管理**: 持久化上下文与常规浏览器混用
